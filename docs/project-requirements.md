# 项目需求文档 — AI Expense Tracker

> **唯一需求真相来源**。任何需求变更必须更新本文档。
> 变更历史见 [iteration-log.md](iteration-log.md)。

---

## 1. 项目愿景

构建一个智能个人财务管理系统，通过**交互友好的 Web 界面 + 后端 API**，帮助用户简单记账、自动分类、了解消费习惯。

## 2. 当前版本：V1.0 MVP

### 2.1 前端 — 用户交互（共 6 个页面）

| ID | 页面 | 功能 | 优先级 |
|----|------|------|--------|
| FE-01 | 登录页 | 邮箱 + 密码登录，登录后跳转首页 | P0 |
| FE-02 | 注册页 | 邮箱 + 密码 + 昵称注册，注册完自动跳转登录 | P0 |
| FE-03 | 首页仪表盘 | 本月收入/支出/结余卡片 + 最近账单列表 | P0 |
| FE-04 | 账单管理页 | 账单列表（搜索/筛选/分页）+ 新增/编辑/删除 | P0 |
| FE-05 | 分类管理页 | 收入/支出分类列表 + 新增/编辑/删除 | P0 |
| FE-06 | 月度统计页 | 按月查询，收入/支出/结余 + 分类支出占比 | P1 |

### 2.2 后端 — 用户模块

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| USR-01 | 用户注册 | 通过邮箱 + 密码创建账户 | P0 |
| USR-02 | 用户登录 | 邮箱 + 密码登录，返回 JWT Token | P0 |
| USR-03 | 密码加密 | 使用 BCrypt 加密存储密码 | P0 |

### 2.3 后端 — 分类模块

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| CAT-01 | 获取分类列表 | 按类型（INCOME/EXPENSE）查询用户分类 | P0 |
| CAT-02 | 创建分类 | 用户自定义收入或支出分类 | P0 |
| CAT-03 | 预设默认分类 | 新用户注册后自动创建默认收支分类 | P1 |

默认支出分类：餐饮、交通、购物、娱乐、住房
默认收入分类：工资、奖金、投资

### 2.4 后端 — 账单模块（核心）

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| TXN-01 | 创建账单 | 新增收入或支出记录 | P0 |
| TXN-02 | 查询账单列表 | 支持按日期范围、分类、类型筛选，分页 | P0 |
| TXN-03 | 修改账单 | 修改金额、描述、分类 | P0 |
| TXN-04 | 删除账单 | 删除指定账单记录 | P0 |
| TXN-05 | 账单归属校验 | 用户只能操作自己的账单 | P0 |

### 2.5 后端 — 统计模块

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| STAT-01 | 月度汇总 | 按月查询收入/支出/结余 | P0 |
| STAT-02 | 分类统计 | 按分类汇总支出/收入金额 | P1 |

### 2.6 后端 — 安全

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| SEC-01 | JWT 认证 | 所有 API（除注册/登录）需携带有效 Token | P0 |
| SEC-02 | Token 过期 | JWT 设置合理过期时间（7天） | P1 |

### 2.7 后端 — AI 智能模块（V2.0）

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| AI-01 | AI 自动分类 | 用户填写账单描述后，AI 自动从已有分类中建议最匹配的分类 | P0 |
| AI-02 | 消费洞察 | 每月 AI 分析消费模式，给出 3-5 条个性化洞察和建议 | P1 |
| AI-03 | 财务报告 | 按需生成月度 AI 财务摘要报告（含与上月对比） | P1 |

## 3. 非功能需求

| ID | 类别 | 描述 |
|----|------|------|
| NFR-01 | 测试 | Service 层单元测试覆盖率 > 80% |
| NFR-02 | 测试 | Controller 层使用 MockMvc 测试 |
| NFR-03 | 测试 | 集成测试使用 TestContainers |
| NFR-04 | API | 统一 RESTful 风格，JSON 格式 |
| NFR-05 | 数据库 | Flyway 管理数据库版本 |
| NFR-06 | 代码 | 分层清晰：Controller → Manager → Service → Mapper → Entity |
| NFR-07 | 设计规范 | 遵循 `docs/development-standards.md` 中的命名、分包、异常、校验规范 |
| NFR-08 | 异常处理 | 统一错误码体系，全局异常处理器拦截 |
| NFR-09 | 参数校验 | JSR-303 注解校验 + Service 层业务校验 |
| NFR-10 | DTO 隔离 | Entity 不暴露到 Controller，使用 DTO 传输 |
| NFR-11 | 多模块 | Maven 多模块设计，模块间通过接口依赖，支持未来拆微服务 |
| NFR-12 | 前端 | 前后端分离，Vue 3 + Element Plus，开发时 Vite 代理到后端 |
| NFR-13 | UI/UX | 登录/注册表单校验友好提示，操作成功/失败有反馈，表格支持分页 |
| NFR-14 | 联调 | 前端开发环境通过 Vite proxy 连接后端，生产环境 Nginx 反向代理或同域部署 |
| NFR-15 | 建表 | 所有表和字段必须添加 COMMENT，索引命名规范 uk_/idx_，统一 utf8mb4 + InnoDB |
| NFR-16 | 配置 | 敏感信息（DB密码等）走环境变量且无默认值；项目级配置可硬编码 |
